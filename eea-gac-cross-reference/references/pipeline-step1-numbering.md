# 步骤一：编号 — 详细规则

## 1. 编号规则

- **格式**: `GAC_{一级功能名称}_{序号}`
- 只编号 X.X 级别的功能子模块标题（如 3.1, 3.2），不编号更深层级
- 每份 EEA 文档独立计数
- `{一级功能名称}` 必须是纯功能名，去掉章节编号前缀

## 2. 编号识别方法

### 阶段 A — 结构探测

1. 用 `docx_utils.get_body_paragraphs(doc)` 获取全部正文段落
2. 遍历每个段落，用正则提取章节编号
3. 确定哪个深度对应"功能子模块"级别

### 阶段 B — 遍历编号

1. 维护 `current_chapter` 和 `seq`
2. 遇到 X.X 级段落 → 追加编号
3. 跳过附录、非需求章节

## 3. 编号后验证

必须用 `keyword_locator.batch_locate()` 验证每个编号出现在正文段落中。

## 4. 必要产物

1. 更新所有 EEA_OUT
2. `EEA_编号总表.md`
3. `步骤1_编号校验.json`

## 5. 校验门禁

`步骤1_编号校验.json.pass = true`（expected_count == actual_count、无重复、无缺号）
